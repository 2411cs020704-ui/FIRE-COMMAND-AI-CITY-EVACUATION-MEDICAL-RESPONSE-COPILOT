================================================================================

SCENARIO SETUP:
- Time: 2:30 PM on a Tuesday
- Location: MRUH Campus, Engineering Block (Building at bottom of hill)
- Weather: Wind from North at 15 km/h, Temperature 32°C
- Population: 500 students in Engineering block, 200 in nearby Science block
- Vulnerable: 10 elderly staff members, 5 people with mobility issues

INCIDENT TRIGGER:
- Fire starts in Engineering Block basement (electrical fault)
- Smoke detected by sensors at 2:31 PM
- System receives alert: "FIRE - Engineering Block - Basement"

PART 8: TECHNICAL INTEGRATION POINTS
================================================================================

How the 3 modules communicate in code:

INCIDENT FLOW:
1. SDG 11 detects incident → stores in database
2. SDG 13 retrieves incident → predicts hazard spread
3. SDG 03 retrieves incident → assesses health risks
4. AI Agent retrieves data from all 3 modules → makes decision
5. Decision → triggers evacuation routes, alerts, resources
6. All 3 modules monitor and update in real-time

DATABASE SCHEMA:
- incidents table: incident_id, type, location, severity, time
- hazard_predictions table: incident_id, prediction_time, affected_zones, spread_speed
- health_risks table: incident_id, vulnerable_groups, medical_resources_needed
- evacuation_routes table: route_id, start, end, waypoints, accessibility_level
- real_time_status table: incident_id, current_time, people_evacuated, status

API ENDPOINTS:
- POST /incidents/report → SDG 11 detects incident
- GET /incidents/{id}/hazard-prediction → SDG 13 predicts spread
- GET /incidents/{id}/health-risks → SDG 03 assesses risks
- POST /evacuation/calculate → AI calculates routes
- GET /evacuation/status → Real-time monitoring
