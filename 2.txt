PART 3: THE AI AGENT ARCHITECTURE (CORE LOGIC)
================================================================================

The AI Agent is the BRAIN of the system. It orchestrates everything.

AGENT FLOW:
1. Receive incident report (voice, text, or sensor)
2. Process through RAG (retrieve relevant knowledge)
3. Query LLM (classify incident, extract data)
4. Execute actions (create incident, calculate routes, allocate resources)
5. Monitor and adapt (update heatmap, send alerts, optimize routes)

AGENT COMPONENTS:

Component 1: INCIDENT PROCESSOR
Purpose: Convert raw input (voice, text) into structured incident data
Input: Voice/text report ("Aag! Fire in Engineering block!")
Output: Structured incident {type: "fire", severity: "critical", location: {x: 0, y: 0}}
Implementation:
  - Use Whisper API for voice transcription
  - Use GPT-4o-mini for incident classification
  - Use regex/NLP for location extraction
  - Use confidence scoring to validate data

Component 2: KNOWLEDGE RETRIEVER (RAG)
Purpose: Retrieve relevant knowledge from campus database
Input: Incident data {type: "fire", location: {x: 0, y: 0}}
Output: Relevant knowledge {nearby_buildings: [...], evacuation_routes: [...]}
Implementation:
  - Query campus zones database
  - Retrieve pre-calculated evacuation routes
  - Retrieve building information
  - Retrieve resource availability
  - Use semantic search for context-aware retrieval

Component 3: DECISION MAKER (LLM)
Purpose: Make intelligent decisions based on incident and knowledge
Input: Incident + Knowledge
Output: Decision {action: "evacuate", priority: "critical", routes: [...]}
Implementation:
  - Use GPT-4o-mini with system prompt
  - Provide context (incident, knowledge, current state)
  - Ask for specific decision (action, priority, routes)
  - Parse structured output (JSON)

Component 4: ACTION EXECUTOR
Purpose: Execute decisions and update system state
Input: Decision {action: "evacuate", routes: [...]}
Output: Updated system state (incident created, alerts sent, routes displayed)
Implementation:
  - Create incident in database
  - Calculate evacuation routes using A*
  - Update heatmap zones
  - Send alerts to users
  - Update admin dashboard

Component 5: MONITOR & ADAPT
Purpose: Monitor incident evolution and adapt responses
Input: Real-time incident updates
Output: Updated routes, alerts, resource allocation
Implementation:
  - Poll incident status every 5 seconds
  - Recalculate routes if incident spreads
  - Update heatmap in real-time
  - Send updated alerts
  - Optimize resource allocation

PART 4: RAG IMPLEMENTATION (KNOWLEDGE LAYER)
================================================================================

RAG = Retrieval-Augmented Generation
Purpose: Give the LLM access to campus-specific knowledge without fine-tuning

KNOWLEDGE BASE:
1. Campus Structure
   - 6 buildings (Engineering, Science, Library, Hostel, Medical, Admin)
   - 10x10 grid system (100 zones)
   - Coordinates for each building
   - Capacity of each building (for sheltering)

2. Evacuation Routes
   - Pre-calculated routes from each building to safe exits
   - Waypoints for each route
   - Estimated evacuation time
   - Capacity constraints

3. Resource Inventory
   - Medical supplies (location, quantity)
   - Food/water (location, quantity)
   - Shelters (location, capacity)
   - Emergency vehicles (location, availability)

4. Incident History
   - Past incidents (type, severity, duration, impact)
   - Lessons learned
   - Recommended responses

RETRIEVAL STRATEGY:
1. Vector embedding: Convert queries to embeddings
2. Semantic search: Find similar knowledge items
3. Ranking: Rank by relevance and recency
4. Filtering: Filter by incident type and location
5. Aggregation: Combine multiple knowledge items

IMPLEMENTATION:
Use simple vector search (no expensive embedding models):
- Convert knowledge to text descriptions
- Use TF-IDF or BM25 for ranking
- Cache results for performance
- Update knowledge base in real-time

      expect(decision.action).toBe('evacuate')
      expect(decision.priority).toBe('critical')
    })

    it('should make medical response decision for medical incident', async () => {
      const incident = {type: 'medical', severity: 'critical', location: {x: 100, y: 100}}
      const knowledge = {buildings: [...], routes: [...]}
      const decision = await makeDecision(incident, knowledge)
      expect(decision.action).toBe('medical')
      expect(decision.priority).toBe('critical')
    })

        {type: 'fire', severity: 'critical', location: {x: 0, y: 0}},

      ]
      const decision = await makeDecision(incidents, knowledge)
      expect(decision.priority).toBe('critical') // Prioritize fire
    })
  })

  // Test 4: Route Calculation
  describe('Pathfinding Engine', () => {
    it('should calculate evacuation routes', async () => {
      const building = {name: 'Engineering', x: 0, y: 0}
      const incident = {type: 'fire', location: {x: 0, y: 0}}
      const routes = await calculateRoutes(building, incident)
      expect(routes.length).toBeGreaterThan(0)
      expect(routes[0].waypoints).toBeDefined()
      expect(routes[0].time).toBeGreaterThan(0)
    })

    it('should avoid incident zones', async () => {
      const building = {name: 'Engineering', x: 0, y: 0}
      const incident = {type: 'fire', location: {x: 0, y: 0}}
      const routes = await calculateRoutes(building, incident)
      // Routes should not pass through incident zone
      for (const route of routes) {
        for (const waypoint of route.waypoints) {
          const distance = Math.sqrt(Math.pow(waypoint.x - incident.location.x, 2) + Math.pow(waypoint.y - incident.location.y, 2))
          expect(distance).toBeGreaterThan(incident.radius)
        }
      }
    })

    it('should optimize for speed', async () => {
      const building = {name: 'Engineering', x: 0, y: 0}
      const incident = {type: 'fire', location: {x: 0, y: 0}}
      const routes = await calculateRoutes(building, incident)
      // First route should be fastest
      expect(routes[0].time).toBeLessThanOrEqual(routes[1].time)
    })

    it('should handle multiple incidents', async () => {
      const building = {name: 'Engineering', x: 0, y: 0}
      const incidents = [
        {type: 'fire', location: {x: 0, y: 0}},
        {type: 'flood', location: {x: 100, y: 100}}
      ]
      const routes = await calculateRoutes(building, incidents)
      // Routes should avoid all incident zones
      expect(routes.length).toBeGreaterThan(0)
    })
  })

    it('should update heatmap when incident created', async () => {
      const incident = {type: 'fire', location: {x: 0, y: 0}}

      expect(heatmap.zones[0].severity).toBe('critical')
      expect(heatmap.zones[1].severity).toBe('warning')

    it('should color-code zones correctly', async () => {
      const incident = {type: 'fire', location: {x: 0, y: 0}}
      const heatmap = await updateHeatmap(incident)
      // Check color progression: red > orange > yellow > green
      for (let i = 0; i < heatmap.zones.length; i++) {
        const zone = heatmap.zones[i]
        const distance = Math.sqrt(Math.pow(zone.x - incident.location.x, 2) + Math.pow(zone.y - incident.location.y, 2))
        if (distance < 50) expect(zone.severity).toBe('critical')
        else if (distance < 100) expect(zone.severity).toBe('warning')
        else expect(zone.severity).toBe('info')
      }
    })

    it('should handle multiple incidents', async () => {
      const incidents = [
        {type: 'fire', location: {x: 0, y: 0}},
        {type: 'flood', location: {x: 100, y: 100}}
      ]
      const heatmap = await updateHeatmap(incidents)
      // Zones should reflect maximum severity from all incidents
      expect(heatmap.zones[0].severity).toBe('critical') // Near fire
      expect(heatmap.zones[50].severity).toBe('warning') // Between incidents
    })
  })

  // Test 6: Alert Generation
  describe('Alert System', () => {
    it('should generate alerts in multiple languages', async () => {
      const incident = {type: 'fire', location: {x: 0, y: 0}}
      const alerts = await generateAlerts(incident)
      expect(alerts.length).toBeGreaterThanOrEqual(3) // English, Hindi, Telugu
      expect(alerts.some(a => a.language === 'en')).toBe(true)
      expect(alerts.some(a => a.language === 'hi')).toBe(true)
      expect(alerts.some(a => a.language === 'te')).toBe(true)
    })

    it('should include actionable information', async () => {
      const incident = {type: 'fire', location: {x: 0, y: 0}}
      const alerts = await generateAlerts(incident)
      for (const alert of alerts) {
        expect(alert.message).toContain('evacuate') // Should mention action
        expect(alert.message).toContain('exit') // Should mention route
      }
    })

    it('should prioritize critical alerts', async () => {
      const incidents = [
        {type: 'fire', severity: 'critical'},
        {type: 'info', severity: 'info'}
      ]
      const alerts = await generateAlerts(incidents)
      // Fire alert should come first
      expect(alerts[0].incident.type).toBe('fire')
    })
  })

  // Test 7: End-to-End Flow
  describe('End-to-End Flow', () => {
    it('should handle complete incident response', async () => {
      // 1. User reports incident
      const input = "Aag! Engineering block mein aag hai!"
      
      // 2. System processes incident
      const incident = await processIncident(input)
      expect(incident.type).toBe('fire')
      
      // 3. System retrieves knowledge
      const knowledge = await retrieveKnowledge(incident)
      expect(knowledge.buildings.length).toBeGreaterThan(0)
      
      // 4. System makes decision
      const decision = await makeDecision(incident, knowledge)
      expect(decision.action).toBe('evacuate')
      
      // 5. System executes action
      const result = await executeAction(decision)
      expect(result.incident).toBeDefined()
      expect(result.routes).toBeDefined()
      expect(result.alerts).toBeDefined()
      
      // 6. System updates heatmap
      const heatmap = await updateHeatmap(incident)
      expect(heatmap.zones[0].severity).toBe('critical')
    })

    it('should respond within 2 seconds', async () => {
      const input = "Aag! Engineering block mein aag hai!"
      const start = Date.now()
      
      const incident = await processIncident(input)
      const knowledge = await retrieveKnowledge(incident)
      const decision = await makeDecision(incident, knowledge)
      const result = await executeAction(decision)
      
      const elapsed = Date.now() - start
      expect(elapsed).toBeLessThan(2000)
    })
  })

  // Test 8: Performance Tests
  describe('Performance', () => {
    it('should handle 100 simultaneous incidents', async () => {
      const incidents = []
      for (let i = 0; i < 100; i++) {
        incidents.push({
          type: 'fire',
          location: {x: Math.random() * 250, y: Math.random() * 250}
        })
      }
      
      const start = Date.now()
      const results = await Promise.all(incidents.map(i => executeAction(i)))
      const elapsed = Date.now() - start
      
      expect(results.length).toBe(100)
      expect(elapsed).toBeLessThan(10000) // Should handle 100 incidents in < 10 seconds
    })

    it('should maintain < 100ms latency at scale', async () => {
      const latencies = []
      for (let i = 0; i < 100; i++) {
        const start = Date.now()
        await retrieveKnowledge({type: 'fire', location: {x: 0, y: 0}})
        latencies.push(Date.now() - start)
      }
      
      const avgLatency = latencies.reduce((a, b) => a + b) / latencies.length
      expect(avgLatency).toBeLessThan(100)
    })
  })
