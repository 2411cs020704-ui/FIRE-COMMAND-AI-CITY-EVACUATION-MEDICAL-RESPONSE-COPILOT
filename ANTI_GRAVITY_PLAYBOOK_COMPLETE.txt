PART 3: PHYSICS ENGINE ARCHITECTURE (CANNON.JS)
================================================================================

The physics engine is the HEART of this system. It simulates real-world forces.

PHYSICS COMPONENTS:

Component 1: GRAVITY SIMULATION
Purpose: Simulate how gravity affects human movement and object behavior
Implementation:
  - Gravity constant: 9.81 m/s² (Earth-like)
  - Direction: Always downward (toward lowest elevation)
  - Effect on evacuation: People move slower uphill, faster downhill
  - Effect on water: Flows downhill, creates new hazards
  - Effect on objects: Buildings, debris, vehicles affected by slope

Gravity Impact on Evacuation:
  - Flat terrain: Movement speed = 1.4 m/s (normal walking)
  - Uphill (5°): Movement speed = 1.0 m/s (28% slower)
  - Uphill (10°): Movement speed = 0.7 m/s (50% slower)
  - Downhill (5°): Movement speed = 1.6 m/s (14% faster)
  - Downhill (10°): Movement speed = 1.8 m/s (28% faster)

Component 2: TERRAIN SYSTEM
Purpose: Represent campus as 3D terrain with elevation
Implementation:
  - Heightmap: 10x10 grid with elevation values
  - Mesh generation: Create 3D terrain from heightmap
  - Collision detection: People/objects collide with terrain
  - Slope calculation: Calculate slope angle at each point
  - Surface friction: Different friction for different terrain types

Terrain Data Structure:
{
  heightmap: [[0, 5, 10, ...], [5, 8, 12, ...], ...],
  width: 250,
  depth: 250,
  maxHeight: 50,
  terrainTypes: {
    "grass": {friction: 0.8, walkable: true},
    "concrete": {friction: 0.6, walkable: true},
    "water": {friction: 0.1, walkable: false},
    "building": {friction: 1.0, walkable: false}
  }
}

Component 3: FORCE SIMULATION
Purpose: Simulate how external forces affect evacuation
Implementation:
  - Wind forces: Can push people off course
  - Water pressure: From floods, pushes people downhill
  - Explosion forces: From fires, push people away
  - Crowd pressure: People pushing each other during evacuation

Force Calculations:
  - Wind force: F = 0.5 * ρ * v² * A (air density, velocity, area)
  - Water pressure: F = ρ * g * h * A (density, gravity, height, area)
  - Explosion force: F = (TNT equivalent) / distance²
  - Crowd force: F = (number of people) * (average force per person)

Component 4: COLLISION DETECTION
Purpose: Detect when people/objects collide with terrain or obstacles
Implementation:
  - Sphere collision: People represented as spheres
  - Mesh collision: Terrain represented as mesh
  - Continuous collision detection: Prevent tunneling through terrain
  - Collision response: Bounce, slide, or stop based on surface

Collision Types:
  - Person vs Terrain: Stop or slide
  - Person vs Building: Stop
  - Person vs Water: Slow down or stop
  - Person vs Other Person: Crowd dynamics

Component 5: REAL-TIME PHYSICS UPDATE
Purpose: Update physics state every frame
Implementation:
  - Simulation step: 60 FPS (16.67 ms per frame)
  - Integrate forces: Update velocity and position
  - Collision detection: Check for collisions
  - Collision response: Handle collisions
  - Constraint solving: Resolve overlaps

Physics Update Loop:
1. Apply forces (gravity, wind, water, explosion)
2. Update velocity (v = v + a * dt)
3. Update position (p = p + v * dt)
4. Detect collisions
5. Resolve collisions
6. Render frame

PART 4: GRAVITY-AWARE PATHFINDING (A* + PHYSICS)
================================================================================

This is where AI meets physics. The pathfinding algorithm accounts for gravity.

PATHFINDING WITH PHYSICS:

Traditional A* Algorithm:
- Cost = distance
- Heuristic = straight-line distance
- Result = shortest path

Gravity-Aware A* Algorithm:
- Cost = distance + elevation change + slope difficulty + time to traverse
- Heuristic = straight-line distance + elevation change
- Result = fastest path considering physics

Cost Function:
cost(node) = distance + (elevation_change * gravity_factor) + (slope * difficulty_factor) + time

Where:
- distance = Euclidean distance between nodes
- elevation_change = |current_elevation - next_elevation|
- gravity_factor = 0.5 (weight for elevation in cost calculation)
- slope = angle of terrain (0-90 degrees)
- difficulty_factor = 1 + (slope / 45) (difficulty increases with slope)
- time = distance / movement_speed(slope)

Movement Speed Calculation:
movement_speed(slope) = base_speed * speed_factor(slope)

Where speed_factor is:
- Flat (0°): 1.0
- Uphill 5°: 0.71
- Uphill 10°: 0.50
- Uphill 15°: 0.35
- Downhill 5°: 1.14
- Downhill 10°: 1.28
- Downhill 15°: 1.43

Heuristic Function:
h(node) = straight_line_distance + elevation_change_estimate

Where elevation_change_estimate = |current_elevation - goal_elevation| * gravity_factor

ALGORITHM PSEUDOCODE:

function gravityAwareAstar(start, goal, terrain, incident) {
  openSet = [start]
  cameFrom = {}
  gScore = {start: 0}
  fScore = {start: heuristic(start, goal, terrain)}
  
  while openSet is not empty {
    current = node in openSet with lowest fScore
    
    if current == goal {
      return reconstructPath(cameFrom, current)
    }
    
    remove current from openSet
    
    for neighbor in getNeighbors(current, terrain) {
      // Skip if neighbor is in incident zone
      if isInIncidentZone(neighbor, incident) {
        continue
      }
      
      // Calculate terrain-aware cost
      slope = calculateSlope(current, neighbor, terrain)
      movementSpeed = calculateMovementSpeed(slope)
      traversalTime = distance(current, neighbor) / movementSpeed
      elevationCost = abs(neighbor.elevation - current.elevation) * GRAVITY_FACTOR
      
      tentativeGScore = gScore[current] + traversalTime + elevationCost
      
      if neighbor not in gScore or tentativeGScore < gScore[neighbor] {
        cameFrom[neighbor] = current
        gScore[neighbor] = tentativeGScore
        fScore[neighbor] = gScore[neighbor] + heuristic(neighbor, goal, terrain)
        
        if neighbor not in openSet {
          add neighbor to openSet
        }
      }
    }
  }
  
  return null // No path found
}

MULTI-OBJECTIVE OPTIMIZATION:

Instead of just finding fastest path, optimize for:
1. Speed (minimize time)
2. Safety (avoid incident zones)
3. Accessibility (avoid steep slopes for elderly/disabled)
4. Capacity (distribute people across multiple routes)

Multi-Objective Cost:
cost = w1*time + w2*safety + w3*accessibility + w4*capacity

Where weights can be adjusted based on incident type:
- Fire: w1=0.5, w2=0.3, w3=0.1, w4=0.1 (prioritize speed and safety)
- Flood: w1=0.3, w2=0.4, w3=0.2, w4=0.1 (prioritize safety and accessibility)
- Medical: w1=0.4, w2=0.2, w3=0.3, w4=0.1 (prioritize accessibility)

PART 5: PHYSICS-BASED HEATMAP (ELEVATION-AWARE)
================================================================================

The heatmap is not just based on distance to incident. It accounts for PHYSICS.

HEATMAP CALCULATION:

Traditional Heatmap:
- Severity = 1 / distance_to_incident
- Result = Circular zones around incident

Physics-Based Heatmap:
- Severity = 1 / (distance_to_incident * physics_factor)
- Physics factor includes: elevation, slope, wind direction, water flow
- Result = Realistic zones that account for terrain

Physics Factors:

1. Elevation Factor:
   - If incident is uphill from zone: severity *= 1.2 (gravity helps spread)
   - If incident is downhill from zone: severity *= 0.8 (gravity opposes spread)

2. Slope Factor:
   - Steep slopes (>15°): severity *= 1.3 (harder to evacuate)
   - Gentle slopes (<5°): severity *= 0.9 (easier to evacuate)

3. Wind Direction Factor:
   - If wind blows toward zone: severity *= 1.5 (smoke/fire spreads)
   - If wind blows away from zone: severity *= 0.6 (smoke/fire disperses)

4. Water Flow Factor (for floods):
   - If zone is downhill from incident: severity *= 1.8 (water flows here)
   - If zone is uphill from incident: severity *= 0.2 (water flows away)

5. Terrain Type Factor:
   - Grass: severity *= 1.0 (normal)
   - Concrete: severity *= 1.2 (fire spreads faster)
   - Water: severity *= 0.5 (water doesn't spread in water)
   - Buildings: severity *= 1.5 (fire spreads through buildings)

HEATMAP CALCULATION ALGORITHM:

function calculatePhysicsBasedHeatmap(incident, terrain, weather) {
  heatmap = {}
  
  for each zone in campusZones {
    // Base severity from distance
    distance = calculateDistance(incident.location, zone.location)
    baseSeverity = 1 / (distance + 1) // +1 to avoid division by zero
    
    // Calculate physics factors
    elevationFactor = calculateElevationFactor(incident, zone, terrain)
    slopeFactor = calculateSlopeFactor(zone, terrain)
    windFactor = calculateWindFactor(incident, zone, weather)
    waterFactor = calculateWaterFactor(incident, zone, terrain)
    terrainFactor = calculateTerrainFactor(zone, terrain)
    
    // Combine factors
    physicsMultiplier = elevationFactor * slopeFactor * windFactor * waterFactor * terrainFactor
    
    // Calculate final severity
    severity = baseSeverity * physicsMultiplier
    
    // Clamp to 0-1 range
    severity = Math.max(0, Math.min(1, severity))
    
    // Color code
    color = getColorForSeverity(severity)
    
    heatmap[zone.id] = {
      severity: severity,
      color: color,
      factors: {elevation: elevationFactor, slope: slopeFactor, wind: windFactor, water: waterFactor, terrain: terrainFactor}
    }
  }
  
  return heatmap
}

ELEVATION FACTOR CALCULATION:

function calculateElevationFactor(incident, zone, terrain) {
  incidentElevation = terrain.getElevation(incident.location)
  zoneElevation = terrain.getElevation(zone.location)
  elevationDifference = zoneElevation - incidentElevation
  
  if elevationDifference > 0 {
    // Zone is uphill from incident
    // Gravity opposes spread, reduce severity
    return 1 - (elevationDifference / 50) * 0.2 // Max 20% reduction
  } else {
    // Zone is downhill from incident
    // Gravity helps spread, increase severity
    return 1 + (abs(elevationDifference) / 50) * 0.2 // Max 20% increase
  }
}

SLOPE FACTOR CALCULATION:

function calculateSlopeFactor(zone, terrain) {
  slope = terrain.getSlope(zone.location) // 0-90 degrees
  
  if slope < 5 {
    return 0.9 // Gentle slope, easier to evacuate
  } else if slope < 15 {
    return 1.0 // Normal slope
  } else if slope < 30 {
    return 1.2 // Steep slope, harder to evacuate
  } else {
    return 1.5 // Very steep, very difficult
  }
}

WIND FACTOR CALCULATION:

function calculateWindFactor(incident, zone, weather) {
  windDirection = weather.windDirection // 0-360 degrees
  incidentToZone = calculateBearing(incident.location, zone.location)
  
  angleDifference = abs(windDirection - incidentToZone)
  if angleDifference > 180 {
    angleDifference = 360 - angleDifference
  }
  
  if angleDifference < 45 {
    // Wind blows toward zone
    return 1.5
  } else if angleDifference > 135 {
    // Wind blows away from zone
    return 0.6
  } else {
    // Wind blows perpendicular
    return 1.0
  }
}

WATER FACTOR CALCULATION (for floods):

function calculateWaterFactor(incident, zone, terrain) {
  incidentElevation = terrain.getElevation(incident.location)
  zoneElevation = terrain.getElevation(zone.location)
  
  if incident.type == "flood" {
    if zoneElevation < incidentElevation {
      // Zone is downhill, water flows here
      return 1.8
    } else if zoneElevation > incidentElevation {
      // Zone is uphill, water flows away
      return 0.2
    } else {
      // Same elevation
      return 1.0
    }
  } else {
    return 1.0 // No water factor for non-flood incidents
  }
}
